# Fake News Prediction System

This project delivers an end-to-end fake news detection workflow based on a Kaggle dataset, a FastAPI backend service, and a React + Flux frontend. The repo ships with training utilities, serving code, and a UI that consumes the prediction API.

## 1. Project Structure

- `backend/`: FastAPI app plus model training pipeline.
- `frontend/`: Vite + React client wired with the Flux architecture (`dispatcher`, `actions`, `stores`).
- `data/`: Place Kaggle CSVs here (ignored by Git).
- `models/`: Serialized artifacts generated by the training script.

## 2. Dataset

1. Pick a Kaggle fake-news dataset (e.g., [`jruvika/fake-news-detection`](https://www.kaggle.com/datasets/jruvika/fake-news-detection)).
2. Download the ZIP locally via the Kaggle CLI:
   ```powershell
   kaggle datasets download -d jruvika/fake-news-detection -p data --unzip
   ```
3. Ensure the combined CSV contains `title`, `text`, and a label column such as `label` (`0` = fake, `1` = real). Update `backend/config.yaml` if your column names differ.

## 3. Model Training

```powershell
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python model_pipeline.py --config config.yaml
```

Artifacts (`tfidf_vectorizer.joblib`, `fake_news_model.joblib`, `label_encoder.joblib`) will be written to `../models/` by default.

## 4. Backend API

Serve predictions via FastAPI:

```powershell
cd backend
.venv\Scripts\activate
uvicorn app:app --reload --port 8000
```

- `GET /health` – readiness probe.
- `POST /predict` – body: `{ "title": "...", "text": "..." }`, returns `{ "label": "...", "score": 0-1 }`.

## 5. Frontend (React + Flux)

```powershell
cd frontend
npm install
npm run dev
```

The UI exposes a simple form, dispatches prediction requests through Flux actions, and displays the model verdict plus confidence.

## 6. Environment Variables

- `FASTAPI_MODEL_DIR`: override default `../models`.
- `FASTAPI_ALLOW_ORIGINS`: comma-separated list of hosts for CORS.
- `VITE_API_BASE_URL`: configure the frontend API base (defaults to `http://localhost:8000`).
